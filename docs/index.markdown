#ソフトウェア工学2026
git/github演習

1. ソフトウェア工学の定義と本質

ソフトウェア工学とは,ソフトウェアの開発,運用,保守に対して,量的なアプローチを適用する学問である.プログラミングではなく,長期間利用されることや大規模開発に対応できることを重視する.また,ソフトウェアは技術や利用環境の変化によって品質が低下するため,継続的な保守が必要である.開発では品質,コスト,納期のバランスを意識することが重要である.

2. 開発の基本原則

開発では,Hookerの7原則を基本的な考え方として学んだ.利用者への価値提供を目的とすること,シンプルな設計を心掛けること,再利用を前提とした設計を行うことが重要である.また,APIは利用者が増えるほど設計者が想定していない使われ方にも依存されるというHyrumの法則についても学んだ.ソフトウェア工学は,システムの大規模化や複雑化,IT人材不足,社会インフラとしての重要性の高まりに対応するために重要である.

3. ソフトウェアライフサイクル

ソフトウェアライフサイクルは,要件定義,設計・実装,テスト,運用・保守の工程から構成される.要件定義ではシステムの目的や機能を明確にし,外部委託ではRFPの基礎となる.設計・実装では仕様に基づいてプログラムを作成し,テストではバグの早期発見と修正を行う.運用・保守では,稼働後もシステムを改善しながら継続的に運用する.

4. プロジェクト管理と開発プロセス

プロジェクトは期限と目的を持つ活動であり,目標から逆算して計画を立てるバックキャスティングの考え方が重要である.規模や工数の見積もりにはステップ数やファンクションポイント法が利用される.開発プロセスには,工程を順番に進めるウォーターフォール型,試作を繰り返すスパイラルモデル,変化に柔軟に対応するアジャイル開発などがある.また,作業を細分化して管理するWBSの活用についても学んだ.

5. CI/CDとGitHub Pages

CI/CDは, ソフトウェア開発を効率化するための仕組みである.GitHub Actionsを利用することで,に自動テストやビルド,デプロイを実行できることを学んだ. 実際にGitHub Pagesを用いて講義内容をまとめたWebページを公開し, CI/CDの流れを体験した.
